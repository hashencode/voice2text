# 录音页完善方案（Office Hours）

## 1. 目标定义

当前 `record` 页已有最小录音能力（开始/停止 + 时长 + 入库），但缺少完整的“录音产品化体验”。本方案目标：

1. 让首次用户 3 秒内理解如何开始录音。
2. 保证录音过程中可控、可感知（状态明确、误触低）。
3. 保证结束后有明确去向（已保存、可继续处理）。
4. 异常场景可恢复、可解释（权限拒绝、系统打断、保存失败）。

## 2. 页面信息架构（单页完成）

页面分 3 个区域，从上到下：

1. 顶部栏：标题「录音」+ 可选说明入口（“录音设置”）。
2. 中央主区：
- 初始态仅显示一个超大圆形按钮（麦克风图标，无文字）。
- 录音态切换为超大计时数字（`MM:SS`），数字下方显示“暂停”“停止”两个按钮。
3. 底部操作区：结束后显示“转文字”“重录”“删除”。

设计原则：初始态极简（只保留一个入口动作），录音态操作聚焦（暂停/停止两动作）。

## 3. 核心状态机

统一状态：

- `idle`（待机）
- `starting`（启动中）
- `recording`（录音中）
- `stopping`（停止并保存中）
- `saved`（已保存，可后续处理）
- `error`（错误）

状态迁移：

1. `idle -> starting -> recording`
2. `recording -> stopping -> saved`
3. 任意状态失败 -> `error`
4. `saved` 可进入 `idle`（重录）或跳转转写流程

约束：

- `starting/stopping` 禁止重复点击。
- `recording` 下返回页面需二次确认（避免误退）。

## 4. 关键交互逻辑

### 4.1 录音开始

- 点“开始录音”后立刻进入 `starting`，按钮 loading。
- 权限未授权时弹窗引导（去系统设置）。
- 成功进入 `recording` 后：中央区域切换为超大计时数字，显示“暂停”“停止”按钮。

### 4.2 录音中

- 顶部或状态区显示“录音中，请勿锁屏/切换通话”。
- 每 200ms 刷新计时数字；暂停后冻结时间显示，再次恢复后继续累加。
- 如果系统中断麦克风（来电/其他音频抢占），立刻结束并提示“录音被系统中断，已尝试保存”。

### 4.3 停止与保存

- 点“停止并保存”后进入 `stopping`，防重复点击。
- 成功：显示“已保存并入库”，进入 `saved`。
- 失败：进入 `error`，给出“重试保存”或“重录”。

### 4.4 保存后动作

- 主按钮切换为“转文字”。
- 次级按钮：
  - “继续录音（重录）”
  - “删除录音”
- 跳转转写时携带 `path` + `sessionId`。

## 5. 异常与边界场景

必须覆盖：

1. 权限拒绝（首次拒绝、已拒绝）
2. 启动失败（native 异常）
3. 停止失败（stop 失败）
4. 保存成功但入库失败（提示并允许重试入库）
5. App 退后台/被杀后的恢复（已有 `useRecordingRecovery` 继续保留）
6. 空路径或损坏文件（阻止后续转写）

## 6. UI 规范（结合当前项目风格）

1. 使用 `DefaultLayout` + `TextX` + `ButtonX`，不引入新设计系统。
2. 初始主按钮使用超大圆形（建议直径 120-160），仅麦克风图标。
3. 录音中计时数字使用大字号（建议 56-72），下方操作按钮使用等宽双按钮布局。
4. 状态颜色：
   - 录音中：`red`
   - 成功：`green`
   - 错误：`destructive`
5. 暗黑模式跟随主题，不使用硬编码白底黑字。
6. 不显示波形，不显示音频电平条。

## 7. 数据与事件

入库字段（已具备）：

- `path`
- `sampleRate`
- `numSamples`
- `durationMs`
- `recordedAtMs`
- `sessionId`

建议增加埋点（后续可接）：

- `record_start`
- `record_stop`
- `record_save_success`
- `record_save_fail`
- `record_interrupted`

## 8. 验收标准（Definition of Done）

1. 录音流程无双击/连点异常。
2. 浅色/深色模式视觉一致。
3. 权限拒绝、系统中断、保存失败都能给出明确下一步。
4. 停止后 1 秒内给出可见结果（成功或失败）。
5. 录音完成后可直达“转文字”，不需要再回首页找文件。

## 9. 实施拆分（建议两阶段）

### Phase A（核心可用，优先）

1. 重构 `record` 页状态机为 `idle/starting/recording/stopping/saved/error`。
2. 新增保存后动作区（转文字/重录/删除）。
3. 补全异常处理与状态文案。

### Phase B（体验增强）

1. 暂停/恢复的可视反馈打磨（按钮态与状态文案）。
2. 系统中断的更细粒度提示。
3. 埋点与性能优化（卡顿监控、长录音稳定性）。

## 10. 推荐决策（Office Hours结论）

RECOMMENDATION: 采用“单页完整状态机 + 保存后直接转写入口”的方案。

原因：这是最小心智负担、最贴近录音主任务、并且能完整覆盖异常场景的实现路径。  
Completeness: 9/10（Phase A 即可达成，Phase B 补齐体验上限）。
